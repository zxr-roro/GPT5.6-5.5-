## 5. 文档输出规范

### 输出位置

- 文档默认输出到**用户当前项目目录**（不是 skill 包目录）
- 文件名格式：`YYYY-MM-DD_[类型]-[目标简称]-report.md`
- 如果用户项目有 `docs/` 目录，优先放在 `docs/` 下

### 输出时机

AI 在以下时机自动调用本 skill 生成文档：

1. 逆向任务完成，已产出核心结论
2. 渗透测试完成，已发现并验证漏洞
3. CTF 题目解出，已拿到 flag
4. 用户明确要求"写一份报告/文档"

### 质量要求

- 所有代码块必须可直接运行或有明确上下文
- 不要有 placeholder/TODO（如果某部分确实未完成，标注"待补充"并说明原因）
- 关键发现必须有证据支撑（命令输出、截图描述、代码片段）
- 复现步骤必须让第三方能独立重现
